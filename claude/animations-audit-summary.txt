Detailed Descriptions
---------------------

1. BUG: toString logic in animation.ts is reversed (cosmetic)
   In animation.ts toString(), the range-printing loop adds a comma on the FIRST
   iteration and then sets first=false so subsequent entries have no separator.
   The logic should be inverted: add the comma for subsequent items, not the first.

2. BUG: toString in animationGroup.ts prints object instead of length
   The line:
     ret += ", animatables length: " + this._animatables;
   concatenates the array object itself. Should be this._animatables.length.

3. BUG: Potential division by zero in animatorAvatar.ts
   The proportionRatio is computed as a ratio of target/source differences on the
   vertical axis. If sourceRootGroundReferenceDiff on the chosen axis is 0, this
   will produce Infinity or NaN. No guard exists for this case.

4. BUG: Vertical axis auto-detection doesn't compare all 3 axes properly
   The algorithm checks Y>X then Z>Y independently. If X=10, Y=5, Z=7: the first
   check fails (verticalAxis stays 0=X), the second check passes (7>5, sets Z),
   even though X=10 is the largest. Need to compare against the current winner.

5. ISSUE: pathCursor.ts throws a string literal instead of an Error
   throw "step size should be less than 1." should be throw new Error(...)
   for proper stack traces and error handling.

6. ISSUE: _registerTargetForLateAnimationBinding is duplicated
   The same function exists as a private method in runtimeAnimation.ts and as an
   exported function RegisterTargetForLateAnimationBinding in animatable.core.ts.
   The runtimeAnimation.ts version is used directly; the exported one is identical
   logic. This could be consolidated.

7. IMPROVEMENT: easing.ts uses hardcoded numeric constants for Pi
   Values like 3.1415926535897931, 6.2831853071795862, 1.5707963267948966 could
   use Math.PI, Math.PI * 2, Math.PI / 2 for clarity.

8. IMPROVEMENT: ElasticEase.easeInCore uses == for number comparison
   if (num == 0) should use strict equality === for consistency.

9. OPTIMIZATION: Interpolation functions allocate new objects every frame
   All interpolation functions (quaternionInterpolateFunction, etc.) return new
   objects every call. matrixInterpolateFunction already supports a result param.
   Extending this pattern would reduce GC pressure. (Not fixed — too invasive.)

10. IMPROVEMENT: animatable.core.ts _animate splice uses indexOf on large array
    Every animation end does a linear scan + splice. Compare with animationGroup's
    stop() which uses a more efficient compaction approach. (Not fixed — behavioral.)

11. IMPROVEMENT: runtimeAnimation.ts constructor mutates shared Animation key array
    If _minFrame !== 0, it splices a frame-0 key into the shared _keys array.
    This mutates the Animation object's data. Works by accident since subsequent
    RuntimeAnimations see _minFrame === 0. (Not fixed — behavioral risk.)

12. OBSERVATION: _fixAnimationGroup in animatorAvatar.ts shadows loop variable
    Inner for-loop uses let i which shadows outer let i. Should use a different
    variable name.

13. IMPROVEMENT: animatorAvatar.ts uses verbose manual iterator patterns
    for (let key = iterator.next(); key.done !== true; ...) could simply use
    for (const skeleton of this.skeletons).
